# LensChecks
Репозиторий, где собраны все компоненты модульной проверки анкет.

## Подготовка к использованию
Перед самым первым запуском необходимо:

Открыть JupyterLab в любой папке, где хочется устновить LensChecks. Перейти в JupyterLab Terminal. Набрать по очереди команды:
1. Клонирование репозитория \
```git clone https://github.com/Ivsoffy/LensChecks.git```
2. Установка библиотеки для зависимостей \
```powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"```
3. Установка зависимостей \
```cd LensChecks```\
```uv sync```
4. Скачивание весов для моделей \
```uv run scripts\download_weights.py```

## Инструкция по использованию
Запуск любого модуля проверки состоит из двух шагов:
- Настройка config.yml
- Запуск start.py


Для получения последней версии кода можно прописать в терминале (находясь в папке LensCheck) две команды:

```
git fetch origin
git reset --hard origin/main
```

### Config.yml
Здесь можно настроить параметры проверки.\
**Общие параметры:**
- module: [1,2,3,4,5] - Выбор модуля для проверки
- input_folder: Директория с проверяемыми файлами. Если параметр пустой, то для 1 модуля по умолчанию просматривается папка module_1/input, а для остальных модулей по умолчанию просматривается папка output, созданная автоматически внутри предыдущего модуля.

**Параметры для разных модулей:**
#### 1 модуль
- save_db_only_without_errors: [True,False] - False, если нужно сохранить анкету для продолжения проверки, даже если в анкете присутствуют ошибки
#### 2 и 4 модули
- folder_past_year: - Директория с анкетами прошлого года для подтягивания кодов и грейдов
- after_fix: [True,False] - Выставляется True после того, как аналитик проверил коды и грейды, чтобы отобразить проверенные уникальные значения на всю таблицу.
#### 5 модуль
- clean: [True,False] - если True, то output 5 модуля попадает сразу в папку Modules/src/results, а output'ы всех остальных модулей очищаются, данные оттуда перемещаются в archive.

### Запуск start.py
Запустить код можно в jupyter terminal: \
```cd src``` \
```uv run start.py```

## Информация по каждому модулю
### Модуль 1
Техническая проверка. В файле исправляются ошибки, данные нормализуются до выпадающих списков, проверяется заполнение обязательных колонок.\
**Input:** Директория с анкетами от клиентов в формате Lens.\
**Output:**
- module_1/output/company_name.xlsx - Excel-файлы с исправленными данными для каждой компании.
- module_1/output/unprocessed/company_name.xlsx - Анкеты, в которых были обнаружены ошибки. Сохраняются в формате Lens с добавлением страницы "Ошибки", на которой перечислены все ошибки. На странице "Данные" все ошибочные строки подсвечены цветом.

### Модуль 2
Проверка и выставление кодов функий. Происходит в 2 этапа. На первом этапе код подтягивает функции с прошлого года и выставляет моделью все, что осталось непроставленным. Аналитик проверяет, вносит правки на листы Prefill и Model. На втором этапе код запускается с параметром ```after_fix: True```, все значения из Prefill и Model подтягиваются в исходную таблицу.

1-й этап:

**Input**: module_1/output/\
**Output**: module_2/output/require_check/company_name.xlsx - Excel-файлы с четырьмя листами:
1. Total Data - все данные с подтянутыми кодами с прошлого года.
2. Prefill - уникальные строки, в которых коды уже были предзаполнены клиентом
3. Model - уникальные строки, в которых коды проставила модель
4. Info - информация о проставленных и отсутствовавших кодах

2-й этап:

**Input**: module_2/output/require_check/company_name.xlsx \
**Output**: module_2/output/company_name.xlsx - Excel-файлы, в которых исправленные аналитиком значения на листах Prefill, Model перенесены в общую таблицу.
### Модуль 3
В этом модуле рассчитываются компенсационные элементы.

**Input**: module_2/output/company_name.xlsx\
**Output**: module_3/output/company_name.xlsx - Excel-файлы с расчитанными компенсационными элементами.
### Модуль 4
Аналогичен модулю 2, но вместо кодов функций проставляются грейды.

### Модуль 5
Финальный модуль. Технические и логические проверки. Подготовка к загрузке в БД.

**Input**: module_4/output/company_name.xlsx\
**Output**: 
- module_5/output/company_name.xlsx - итоговые анкеты
- module_5/output/unprocessed/company_name.xlsx - анкеты, в которых были обнаружены ошибки. Сохраняются в формате Lens с добавлением страницы "Ошибки", на которой перечислены все ошибки. На странице "Данные" все ошибочные строки подсвечены цветом.
